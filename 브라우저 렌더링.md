# 브라우저 렌더링

> 브라우저 렌더링 과정을 설명하시오

1. 로컬 DNS 캐시를 확인하여 해당 도메인의 IP 주소를 찾습니다.
2. 캐시에 없다면, url의 탑레벨도메인(TLD)을 통해 가장 가까운 DNS 서버에 요청을 보냅니다.
3. DNS 서버에서 url에 해당하는 ip 주소를 응답 받습니다.
4. 변환된 ip 주소의 웹 서버에 get 요청을 보냅니다.
5. 해당 주소에서 html문서를 응답받습니다.
6. 응답받은 문서에서 html 코드를 통해 DOM tree를 생성합니다.
7. 응답받은 문서에서 css 코드가 있는 경우 CSSOM tree를 생성합니다.
8. 생성된 DOM tree와 CSSOM tree를 결합하여 렌더 트리를 생성합니다.
9. `(Layout)` 생성된 렌더 트리에 포함된 각 노드의 위치와 크기를 계산하여 배치합니다.
10. `(Paint)` 배치가 완료된 각 요소에 대한 레이어를 생성합니다.
11. `(Composition)` 여러 레이어가 적절하게 결합되어 화면에 최종적으로 렌더링되는 모습을 구성하여, 브라우저에 노출되게 됩니다.
